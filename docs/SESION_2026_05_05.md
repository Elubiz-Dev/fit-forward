## 🚀 Solución Definitiva: EAS Build (próxima sesión)

### El problema actual
El login con Google **depende de la IP del WiFi** (`exp://192.168.40.107:8081`). Esto significa:
- Si cambias de red WiFi, deja de funcionar.
- Tu compañero de desarrollo tiene una IP diferente y no puede usar Google login.
- No sirve para producción (Play Store).

### La solución: EAS Development Build

En vez de usar Expo Go (que depende de la IP), se genera un **build de desarrollo propio** que usa el scheme `fitgo://` definido en `app.json`. Este scheme es permanente y no depende de ninguna red.

#### Comparativa

| | Expo Go | EAS Development Build |
|---|---|---|
| Depende de IP WiFi | ✅ Sí | ❌ No |
| Google/Facebook login | ⚠️ Complicado | ✅ Funciona perfecto |
| El bro puede usarlo | ⚠️ Solo con su IP | ✅ Sí, siempre |
| Play Store ready | ❌ No | ✅ Sí |

#### Pasos para implementarlo (próxima sesión)

1. **Instalar EAS CLI:**
   ```bash
   npm install -g eas-cli
   eas login
   ```

2. **Configurar EAS en el proyecto:**
   ```bash
   eas build:configure
   ```

3. **Crear el build de desarrollo para Android:**
   ```bash
   eas build --profile development --platform android
   ```
   Genera un `.apk` que se instala en los celulares de todos los devs.

4. **Cambiar la Site URL en Supabase** de la IP local a:
   ```
   fitgo://
   ```

5. **Agregar a las Redirect URLs de Supabase:**
   ```
   fitgo://*
   ```

> ✅ Con esto, tanto tú como tu bro pueden probar Google y Facebook login desde sus celulares sin depender de ninguna IP ni red WiFi.
